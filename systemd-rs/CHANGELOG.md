### Fixed

- **3-way RwLock deadlock in PID table** ‚Äî extracted `pid_table` to `Arc<Mutex<PidTable>>` so the signal handler can update entries without the RuntimeInfo read lock, breaking a deadlock between activation threads, control handler, and exit handler
- **Socket activation `DependencyError` handling** ‚Äî unsatisfied `After=` dependencies silently deferred instead of logged as ERROR
- **Symlink-aware unit file discovery** ‚Äî `symlink_metadata()` instead of `entry.metadata()` for NixOS store symlink chains
- **Udev rules path for systemd-rs overlay** ‚Äî `udevRulesOverride` package ensures correct `systemctl` path in udev `RUN+=` actions
- **PAM authentication race** ‚Äî proper `/run/wrappers` mount ordering and Type=idle deferral

### Changed

- **`systemd-udevd` refactored into library + binary** ‚Äî extracted daemon logic into `systemd_udevd` library crate (`src/lib.rs`) with public API (constants, `UEvent`, `RuleSet`, `RuleResult`, `process_rules`, `glob_match`, `open_uevent_socket`, etc.), thin `src/main.rs` binary entry point; `udevadm` now depends on the `systemd-udevd` library crate, eliminating duplicated constants (`CONTROL_SOCKET_PATH`, `DB_DIR`, `TAGS_DIR`, `QUEUE_FILE`, `RULES_DIRS`), `open_uevent_socket()`, and `glob_match` implementations; `udevadm test` enhanced to run actual rule simulation via `RuleSet::load()` + `process_rules()` instead of just displaying device properties; removed unused `nix`, `serde`, `serde_json`, `tempfile` dependencies from `udevadm`; added `pkg-config` to `default.nix` nativeBuildInputs
- **`systemctl` flag handling** ‚Äî strips `--no-block`, `--quiet`, `--force`, `--no-pager`, `--no-ask-password`, `--system`, `-a`, `-q`, `-f`, `-l`, `-t`, `-p` before sending commands to PID 1
- **`systemctl` command aliases** ‚Äî `poweroff`/`reboot`/`halt` ‚Üí `shutdown`, `daemon-reload` ‚Üí `reload`, `condrestart`/`force-reload` ‚Üí `try-restart`
- **`try-restart`/`reload-or-restart`/`is-active`/`is-enabled`/`is-failed`** added to PID 1 control handler
- **`/etc/mtab` symlink** ‚Äî PID 1 creates `/etc/mtab ‚Üí ../proc/self/mounts` (fixes "failed to update userspace mount table" warnings)
- **VFS mount safety nets** ‚Äî PID 1 early setup ensures `/proc`, `/sys`, `/dev`, `/dev/shm`, `/dev/pts`, `/run` are mounted

### Removed

- **Noisy fork-child debug output** ‚Äî removed `write_to_stderr` calls between `fork()` and `exec()` that produced two lines of noise per service start
