# Changelog

All notable changes to systemd-rs are documented in this file.

## Unreleased

### Added

- **`systemd-nspawn` — lightweight namespace container runtime** — new crate implementing the core of `systemd-nspawn(1)` for spawning commands or full OS boots in isolated Linux namespace containers; **container creation** via `fork()` + `unshare()` with PID, mount, UTS, and IPC namespaces; `--private-network` adds CLONE_NEWNET; `--private-users` / `-U` adds CLONE_NEWUSER with automatic `uid_map`/`gid_map`/`setgroups` setup; **essential API filesystem setup** mounts `/proc`, `/sys` (read-only bind), `/dev` (tmpfs with bind-mounted null/zero/full/random/urandom/tty + fd/stdin/stdout/stderr/ptmx symlinks), `/dev/pts` (devpts newinstance), `/dev/shm`, `/dev/mqueue`, `/run`, `/tmp`; **`--boot` mode** discovers init binary (`usr/lib/systemd/systemd`, `sbin/init`, etc.) and execs it inside the container; **`--directory=PATH`** specifies container root; **`--machine=NAME`** with auto-derivation from directory/image path and sanitization (alphanumeric/dot/underscore/dash, max 64 chars); **bind mounts** via `--bind=SRC[:DST[:OPTS]]` / `--bind-ro` (recursive by default, `norbind` option); **`--tmpfs=PATH[:OPTS]`** and **`--overlay`** / **`--overlay-ro`** mount specifications; **capability bounding** with 41 Linux capabilities (CAP_ prefix handling, case-insensitive, `all` keyword, comma-separated lists); default set of 23 capabilities matching real systemd-nspawn; enforcement via `prctl(PR_CAPBSET_DROP)`; `--capability=CAP` to add, `--drop-capability=CAP` to remove; **`--as-pid2`** stub init mode; **`--user=USER`** (numeric UID or name lookup via `getpwnam` with `setuid`/`setgid`); **`--chdir=PATH`**; **`--setenv=VAR=VALUE`** / **`-E`** (with host environment inheritance for bare variable names); **`--volatile[=yes|state|overlay|no]`**; **`--resolv-conf=MODE`** (15 modes including off/copy-host/copy-stub/bind-host/bind-stub/delete/auto); **`--timezone=MODE`** (off/copy/bind/symlink/delete/auto); **`--link-journal=MODE`** (no/host/try-host/guest/try-guest/auto); **`--console=MODE`** (interactive/read-only/passive/pipe/autopipe); **`--port=PROTO:HOST:CONTAINER`** / **`-p`** port forwarding specification parsing; **`--hostname=NAME`** and **`--uuid=UUID`** container identity; **`--register=BOOL`** machined registration/unregistration via JSON control socket protocol; **`--keep-unit`**, **`--no-new-privileges`** (`PR_SET_NO_NEW_PRIVS`), **`--read-only`**, **`--notify-ready`**, **`--kill-signal=SIG`**, **`--settings=BOOL`**, **`--oci-bundle=PATH`** (detected, not yet implemented), **`--property=NAME=VALUE`**; `pivot_root` with `chroot` fallback; clean container environment (`container=systemd-nspawn`, `container_uuid`, minimal `PATH`); parent-child synchronization via pipe pair; `waitpid` with proper exit code propagation (normal exit, signal death → 128+sig); 130 new unit tests covering capability parsing/display/numbering/list-parsing, bind mount parsing, port forward parsing, all mode enums (volatile/resolv-conf/timezone/link-journal/console) with roundtrip tests, machine name sanitization/derivation, capability computation (default/add/drop/duplicate), comprehensive argument parsing (50+ individual option tests, combined scenarios, positional commands, `--` separator), root resolution, init/shell discovery, directory creation, sync pipe I/O

### Fixed

- **`parse_timeout()` now supports milliseconds and microseconds** — previously `parse_timeout("50ms")` panicked because the `s`-suffix branch tried to parse `"50m"` as a number; rewritten to handle full systemd timespan syntax: `us`/`usec` (microseconds), `ms`/`msec` (milliseconds), `s`/`sec` (seconds), `min`/`minute`/`minutes`, `h`/`hr`/`hrs` (hours), and compound expressions (e.g. `1min 30s`); result precision changed from seconds to microseconds (`Duration::from_micros()`) to preserve sub-second accuracy; graceful handling of unparsable tokens (skipped instead of panicking); fixes `CPUQuotaPeriodSec=`, `MemoryPressureThresholdSec=`, and other timespan directives that use sub-second values

- **`systemd-notify` test race condition** — `test_resolve_notify_socket_present` and `test_resolve_notify_socket_missing` were racing on the global `NOTIFY_SOCKET` env var when run in parallel; fixed by serializing the two tests with a shared `Mutex`
- **`cgroup2::thaw()` wrote wrong value** — `thaw()` incorrectly wrote "1" (which means freeze) instead of "0" (thaw); fixed to write "0" so cgroup thaw actually works
- **3-way RwLock deadlock in PID table** — extracted `pid_table` to `Arc<Mutex<PidTable>>` so the signal handler can update entries without the RuntimeInfo read lock, breaking a deadlock between activation threads, control handler, and exit handler
- **Socket activation `DependencyError` handling** — unsatisfied `After=` dependencies silently deferred instead of logged as ERROR
- **Symlink-aware unit file discovery** — `symlink_metadata()` instead of `entry.metadata()` for NixOS store symlink chains
- **Udev rules path for systemd-rs overlay** — `udevRulesOverride` package ensures correct `systemctl` path in udev `RUN+=` actions
- **PAM authentication race** — proper `/run/wrappers` mount ordering and Type=idle deferral
- **`udevadm` multi-call dispatch for NixOS initrd** — when invoked as `systemd-udevd` (via symlink, as NixOS initrd does: `systemd-udevd -> udevadm`), `udevadm` now detects the invocation name and delegates to `systemd_udevd::run_daemon()` instead of parsing the udevadm CLI; this fixes a boot failure where the initrd's `systemd-udevd --daemon` invocation hit clap's "unexpected argument '--daemon'" error, preventing udev from running and blocking root device discovery
