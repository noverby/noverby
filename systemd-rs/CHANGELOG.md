### Fixed

- **3-way RwLock deadlock in PID table** тАФ extracted `pid_table` to `Arc<Mutex<PidTable>>` so the signal handler can update entries without the RuntimeInfo read lock, breaking a deadlock between activation threads, control handler, and exit handler
- **Socket activation `DependencyError` handling** тАФ unsatisfied `After=` dependencies silently deferred instead of logged as ERROR
- **Symlink-aware unit file discovery** тАФ `symlink_metadata()` instead of `entry.metadata()` for NixOS store symlink chains
- **Udev rules path for systemd-rs overlay** тАФ `udevRulesOverride` package ensures correct `systemctl` path in udev `RUN+=` actions
- **PAM authentication race** тАФ proper `/run/wrappers` mount ordering and Type=idle deferral
- **`udevadm` multi-call dispatch for NixOS initrd** тАФ when invoked as `systemd-udevd` (via symlink, as NixOS initrd does: `systemd-udevd -> udevadm`), `udevadm` now detects the invocation name and delegates to `systemd_udevd::run_daemon()` instead of parsing the udevadm CLI; this fixes a boot failure where the initrd's `systemd-udevd --daemon` invocation hit clap's "unexpected argument '--daemon'" error, preventing udev from running and blocking root device discovery

### Changed

- **Timer scheduler upgraded to full calendar expression evaluation** тАФ replaced the shorthand-only `parse_calendar_shorthand()` function (which only handled `hourly`, `daily`, `weekly`, etc.) with `CalendarSpec::parse()` + `CalendarSpec::next_elapse()` for full `OnCalendar=` support; complex expressions like `*-*-* 06:00:00`, `Mon..Fri *-*-* 09:00`, `*-*-1,15 12:00:00`, and `*-*-* */2:00:00` now fire correctly; timer directive coverage increased from 0% to 64% (9 of 14 directives supported)
- **`systemd-analyze calendar` uses shared `CalendarSpec`** тАФ removed local `CalendarSpec`, `normalize_calendar_expression()`, and `is_day_of_week()` implementations from the analyze crate; now uses the shared `libsystemd::calendar_spec::CalendarSpec` with full `next_elapse` display showing next elapse time(s) with weekday labels and relative "from now" duration
- **`systemd-udevd` refactored into library + binary** тАФ extracted daemon logic into `systemd_udevd` library crate (`src/lib.rs`) with public API (constants, `UEvent`, `RuleSet`, `RuleResult`, `process_rules`, `glob_match`, `open_uevent_socket`, etc.), thin `src/main.rs` binary entry point; `udevadm` now depends on the `systemd-udevd` library crate, eliminating duplicated constants (`CONTROL_SOCKET_PATH`, `DB_DIR`, `TAGS_DIR`, `QUEUE_FILE`, `RULES_DIRS`), `open_uevent_socket()`, and `glob_match` implementations; `udevadm test` enhanced to run actual rule simulation via `RuleSet::load()` + `process_rules()` instead of just displaying device properties; removed unused `nix`, `serde`, `serde_json`, `tempfile` dependencies from `udevadm`; added `pkg-config` to `default.nix` nativeBuildInputs
- **`systemctl` flag handling** тАФ strips `--no-block`, `--quiet`, `--force`, `--no-pager`, `--no-ask-password`, `--system`, `-a`, `-q`, `-f`, `-l`, `-t`, `-p` before sending commands to PID 1
- **`systemctl` command aliases** тАФ `poweroff`/`reboot`/`halt` тЖТ `shutdown`, `daemon-reload` тЖТ `reload`, `condrestart`/`force-reload` тЖТ `try-restart`
- **`try-restart`/`reload-or-restart`/`is-active`/`is-enabled`/`is-failed`** added to PID 1 control handler
- **`/etc/mtab` symlink** тАФ PID 1 creates `/etc/mtab тЖТ ../proc/self/mounts` (fixes "failed to update userspace mount table" warnings)
- **VFS mount safety nets** тАФ PID 1 early setup ensures `/proc`, `/sys`, `/dev`, `/dev/shm`, `/dev/pts`, `/run` are mounted

### Removed

- **Noisy fork-child debug output** тАФ removed `write_to_stderr` calls between `fork()` and `exec()` that produced two lines of noise per service start
