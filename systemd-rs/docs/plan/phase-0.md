# Phase 0 — Foundation (Workspace & Core Library)

Restructure the existing codebase into a Cargo workspace and extract shared functionality into `libsystemd`:

- ✅ **Unit file parser** — complete INI-style parser with all systemd extensions (line continuation, quoting rules, specifier expansion `%i`, `%n`, `%N`, `%p`, `%u`, `%U`, `%h`, `%s`, `%m`, `%b`, `%H`, `%v`, `%t`, etc.)
- ✅ **Dependency graph engine** — topological sort with cycle detection, transaction model for atomic start/stop operations
- ✅ **D-Bus protocol** — pure Rust via the `zbus` crate (migrated from C `libdbus`); all daemon D-Bus interfaces use `#[zbus::interface]` with deferred registration pattern
- ✅ **sd_notify protocol** — full notify socket implementation with credential passing and fd store
- ✅ **Journal binary format** — custom append-only binary format (`JRNL_RS`) with 64-byte file header, framed entries (leading/trailing length for bidirectional iteration), rotation, and vacuuming; `JournalStorage` manages multi-file storage with configurable size/count/disk limits; `JournalReader` provides filtered, seekable iteration with `FieldMatch` criteria (exact field, priority, timestamp range) and `SeekPosition` (head, tail, cursor, after-cursor, realtime timestamp, sequence number); cursor strings encode file ID, seqnum, boot ID, monotonic/realtime timestamps; export format writer (`to_export_format`) and parser (`from_export_format`, `parse_export_entries`) for `systemd-journal-export(5)` interop including binary field support; `export_all()` for bulk export; convenience methods `read_filtered()`, `read_from_cursor()`, `read_from_realtime()`; cursor parsing helpers (`parse_cursor_realtime`, `parse_cursor_boot_id`); journald collects from native socket, syslog, stdout streams, and `/dev/kmsg` with rate limiting, priority filtering, and forwarding; journalctl supports all output formats (short, short-iso, short-precise, short-monotonic, verbose, json, json-pretty, cat, export), boot detection, field enumeration, grep, follow mode, and full filtering; 78+ storage/entry tests
- ✅ **Specifier expansion** — full `specifier` module in `libsystemd` with `SpecifierContext` (system and user variants) and `resolve_specifiers()` supporting all documented `systemd.unit(5)` specifiers: unit identity (`%n`, `%N` with unescaping, `%p`, `%P` with unescaping, `%i`, `%I` with unescaping, `%f` unescaped instance/prefix with leading `/`, `%j`/`%J` first dash-component), user/group (`%u`, `%U`, `%h`, `%s`, `%g`, `%G`), system identity (`%m` machine ID, `%b` boot ID, `%H` hostname, `%l` short hostname, `%v` kernel release, `%a` architecture), directories (`%t` runtime, `%T` tmp, `%V` var-tmp, `%E` config, `%C` cache, `%S` state, `%L` log), credentials (`%d`), os-release (`%o` ID, `%w` VERSION, `%q` VERSION_ID, `%M` PRETTY_NAME, `%A` IMAGE_VERSION, `%B` BUILD_ID, `%W` VARIANT_ID), and `%%` literal percent; `resolve_specifiers_auto()` for automatic instance extraction; `has_unresolved_specifiers()` for template cross-reference detection; os-release parser with `OnceLock` caching; `directory_deps.rs` delegates to the new module via lazily-initialized `SpecifierContext`; 64 specifier tests
- ✅ **Unit name handling** — escaping, unescaping, template instantiation, unit type detection; 19 tests
- ✅ **Configuration parsing** — `/etc/systemd/system.conf`, `/etc/systemd/user.conf`, and environment generators; standard unit search paths (`/etc/systemd/system`, `/run/systemd/system`, `/usr/lib/systemd/system`, `/lib/systemd/system`) with NixOS package-relative directory auto-discovery; PATH augmentation from unit directories
- ✅ **Credential management** — `ImportCredential=` fully implemented (glob-matching from system credential stores), `LoadCredential=` implemented (absolute and relative paths, directory loading), `SetCredential=` implemented (inline data with colon-preserving split), `LoadCredentialEncrypted=` and `SetCredentialEncrypted=` now decrypt at runtime using AES-256-GCM with host key or null key (graceful fallback to writing as-is if decryption fails); credential directory created at `/run/credentials/<unit>/` with correct ownership and 0o700/0o400 permissions; `CREDENTIALS_DIRECTORY` env var set; priority ordering matches systemd (SetCredential < LoadCredential < ImportCredential); `systemd-creds` CLI tool provides encrypt/decrypt with host key (AES-256-GCM), list, cat, setup, and TPM2 detection; missing: TPM2 sealing, host+tpm2 combined mode
- ✅ **Exec section near-complete** — 145 of 147 `systemd.exec(5)` directives supported (99%); logging (`SyslogFacility=`, `SyslogLevel=`, `SyslogLevelPrefix=`, `LogLevelMax=`, `LogRateLimitIntervalSec=`, `LogRateLimitBurst=`, `LogFilterPatterns=`, `LogNamespace=`), CPU scheduling (`CPUSchedulingPolicy=`, `CPUSchedulingPriority=`, `CPUSchedulingResetOnFork=`, `CPUAffinity=`, `NUMAPolicy=`, `NUMAMask=`), root filesystem (`RootDirectory=`, `RootImage=`, `RootImageOptions=`, `RootHash=`, `RootHashSignature=`, `RootVerity=`, `RootEphemeral=`, `MountAPIVFS=`, `ExtensionDirectories=`, `ExtensionImages=`, `MountImages=`, `BindLogSockets=`), namespace isolation (`PrivateIPC=`, `PrivatePIDs=`, `IPCNamespacePath=`, `NetworkNamespacePath=`), security (`SecureBits=`, `Personality=`, `SELinuxContext=`, `AppArmorProfile=`, `SmackProcessLabel=`, `KeyringMode=`, `NoExecPaths=`, `ExecPaths=`, `CoredumpFilter=`, `RestrictFileSystems=`), syscall auditing (`SystemCallLog=` — same syntax as `SystemCallFilter=`, parsed and stored; runtime seccomp audit enforcement pending), misc (`TimerSlackNSec=`, `StandardInputText=`, `StandardInputData=`, `SetLoginEnvironment=`) — all parsed and stored; runtime enforcement pending for most; partial: `LoadCredentialEncrypted=`, `SetCredentialEncrypted=` (parsed and functional with host/null key, TPM2 decryption not implemented); 25 new tests for `SystemCallLog=` and `RestrictFileSystems=`
- ✅ **sd_notify protocol** — `READY=1`, `STATUS=`, `MAINPID=` (updates tracked PID), `WATCHDOG=1` (timestamps recorded for WatchdogSec= enforcement), `WATCHDOG=trigger` (immediate watchdog action), `RELOADING=1` (sets reloading state, clears ready), `STOPPING=1` (sets stopping state), `WATCHDOG_USEC=` (dynamic timeout override — updates watchdog interval and resets ping timestamp), `ERRNO=` (stores last errno for status display), `BUSERROR=` (stores D-Bus error name), `EXIT_STATUS=` (stores exit status string), `MONOTONIC_USEC=` (stores monotonic timestamp), `INVOCATION_ID=` (stores/verifies invocation ID); `READY=1` after `RELOADING=1` clears reload state; **`FDSTORE=1`** with SCM_RIGHTS via `recvmsg()` — stores file descriptors per-service respecting `FileDescriptorStoreMax=`, passed back on restart via `LISTEN_FDS`/`LISTEN_FDNAMES`; **`FDSTOREREMOVE=1`** removes stored FDs by name; **`FDNAME=`** labels stored FDs (default "stored"); **`NOTIFYACCESS=`** runtime override of `NotifyAccess=` — parses `none`/`main`/`exec`/`all` (case-insensitive), stores as `Service::notify_access_override`, `effective_notify_access()` resolves override-then-config precedence, watchdog enforcement and `systemctl show` use effective value; all new fields exposed in `systemctl show` output (`StatusErrno`, `StatusBusError`, `StatusExitStatus`, `InvocationID`, `WatchdogUSec`, `NFileDescriptorStore`, `FileDescriptorStoreMax`); 84 unit tests
- ✅ **Timespan parsing** — `parse_timespan()` and `parse_timeout()` implement full `systemd.time(7)` duration syntax: `us`/`usec` (microseconds), `ms`/`msec` (milliseconds), `s`/`sec`/`second`/`seconds` (seconds), `min`/`minute`/`minutes`/`m` (minutes), `h`/`hr`/`hrs`/`hour`/`hours` (hours), `d`/`day`/`days`, `w`/`week`/`weeks`, `month`/`months`, `y`/`year`/`years`; compound expressions (`1min 30s`, `2h 30min`); bare numbers as seconds; floating-point values; `infinity` keyword; used by all timeout directives (`TimeoutStartSec=`, `TimeoutStopSec=`, `WatchdogSec=`, `RestartSec=`, etc.)
- ✅ **Calendar spec parser/evaluator** — full `systemd.time(7)` calendar expression implementation in `calendar_spec.rs`; shorthands (`minutely`, `hourly`, `daily`, `weekly`, `monthly`, `yearly`, `quarterly`); full date-time expressions (`*-*-* 06:00:00`, `Mon *-*-* 00:00:00`); weekday filters with ranges (`Mon..Fri`); day/month/hour/minute/second lists and ranges; repetitions (`*/N`, `start/step`, `start..end/step`); `CalendarSpec::next_elapse()` computes next matching wall-clock time after a reference point; used by timer scheduler for `OnCalendar=` and by `systemd-analyze calendar`; 81 tests
- ✅ **Condition/Assert evaluation** — 28 condition types with full runtime checks: filesystem (`ConditionPathExists=`, `ConditionPathIsDirectory=`, `ConditionFileNotEmpty=`, `ConditionFileIsExecutable=`, `ConditionDirectoryNotEmpty=`, `ConditionPathIsReadWrite=`, `ConditionPathIsMountPoint=`, `ConditionPathIsEncrypted=`, `ConditionPathIsSymbolicLink=`), virtualization detection (`ConditionVirtualization=` — VM/container/technology-specific detection via DMI, device tree, cgroup, container env), capabilities (`ConditionCapability=` with bounding set check), kernel (`ConditionKernelCommandLine=`, `ConditionKernelModuleLoaded=`), system (`ConditionArchitecture=`, `ConditionFirmware=`, `ConditionHost=` with machine-id and hostname matching, `ConditionEnvironment=`), resources (`ConditionMemory=` with comparison operators, `ConditionCPUs=` with range expressions, `ConditionCPUFeature=`), security (`ConditionSecurity=`), power (`ConditionACPower=`), boot (`ConditionFirstBoot=`, `ConditionNeedsUpdate=`), cgroup (`ConditionControlGroupController=`), os-release (`ConditionOSRelease=` with `=`/`!=`/`<`/`<=`/`>`/`>=` operators), user/group (`ConditionUser=`, `ConditionGroup=`); all conditions support negation (`!` prefix); same set available as `Assert*` directives
- ✅ **Timer scheduler** — background `timer-scheduler` thread in `timer_scheduler.rs`; checks all active `.timer` units every 15 seconds; evaluates monotonic triggers (`OnActiveSec=`, `OnBootSec=`, `OnStartupSec=`, `OnUnitActiveSec=`, `OnUnitInactiveSec=`) relative to boot instant and last-fired timestamps; evaluates `OnCalendar=` expressions via `CalendarSpec::next_elapse()`; fires associated service unit via control interface when a timer elapses; tracks per-timer `last_fired` timestamps to prevent re-firing within the same period; 2-second startup delay to avoid premature OnBootSec=0 firing; 21 tests
- ✅ **Path watcher** — background `path-watcher` thread in `path_watcher.rs`; inotify-based filesystem monitoring for `.path` units; per-condition watch strategy (`PathExists=`/`PathExistsGlob=` watch parent for `IN_CREATE`/`IN_MOVED_TO`; `PathChanged=` watches for `IN_ATTRIB`/`IN_CLOSE_WRITE`/`IN_CREATE`/`IN_DELETE`/`IN_MOVED_FROM`/`IN_MOVED_TO`/`IN_DELETE_SELF`/`IN_MOVE_SELF`; `PathModified=` watches for `IN_CLOSE_WRITE`/`IN_DELETE_SELF`/`IN_MOVE_SELF`; `DirectoryNotEmpty=` watches for `IN_CREATE`/`IN_MOVED_TO`); `PathInotify` watch manager with multi-entry-per-WD support (handles inotify WD reuse); automatic parent-directory watching for non-existent paths with upgrade to direct watch when target appears; `poll(2)` on inotify fd with 500ms timeout; periodic 10-second full-scan safety net for network filesystems; graceful fallback to pure polling when inotify is unavailable; `IN_IGNORED` handling for automatic watch re-establishment; dynamic watch synchronization for newly activated/deactivated path units; glob matching; `MakeDirectory=` support; 44 tests
- ✅ **Watchdog enforcement** — background `watchdog` thread in `watchdog.rs`; periodically scans running services with `WatchdogSec=` configured; effective timeout is `WATCHDOG_USEC=` dynamic override (if service sent one via `sd_notify`) otherwise static `WatchdogSec=`; detects unresponsive services that have not sent `WATCHDOG=1` within the timeout; `WATCHDOG=trigger` clears `watchdog_last_ping` for immediate timeout; sends configured `WatchdogSignal=` (default `SIGABRT`) to main PID; sets `watchdog_timeout_fired` flag so exit handler correctly applies `Restart=on-watchdog`/`Restart=on-failure`/`Restart=on-abnormal` policies; watchdog state cleared on service (re)start; `READY=1` initializes watchdog reference timestamp for notify-type services; 17 tests
- ✅ **Socket activation** — background `socket-activation` thread in `socket_activation.rs`; monitors all active `.socket` unit file descriptors via `select(2)`; when data arrives on a socket, finds the associated service via the socket's `Service=` list (primary) or reverse scan of service `Sockets=` field (fallback); triggers service activation with `ActivationSource::SocketActivation`; passes socket fds to service via `LISTEN_FDS`/`LISTEN_FDNAMES`; supports all socket types: stream/datagram/sequential-packet TCP/UDP (`ListenStream=`, `ListenDatagram=`), Unix domain sockets (`ListenStream=`/`ListenDatagram=` with path), FIFOs (`ListenFIFO=`), netlink sockets (`ListenNetlink=`), special files (`ListenSpecial=`); socket options (`SocketMode=`, `DirectoryMode=`, `PassCredentials=`, `PassSecurity=`, `Accept=`, `MaxConnections=`, `Backlog=`, `KeepAlive=`, `FreeBind=`, `Transparent=`, `ReusePort=`, etc.)
- ✅ **Service lifecycle management** — `services/` module; fork/exec via `exec_helper` child process (shared memory IPC for status reporting); `Service::start()` with `ExecStartPre=`/`ExecStartPost=`/`ExecStopPost=` command sequencing; `run_cmd()` with configurable timeouts; `kill()` with `KillMode=` (control-group, process, mixed, none); `KillSignal=`, `RestartKillSignal=`, `FinalKillSignal=`, `WatchdogSignal=`; `kill_all_remaining_processes()` for cgroup cleanup; `service_exit_handler` background thread with restart policy (`Restart=no`/`always`/`on-success`/`on-failure`/`on-abnormal`/`on-abort`/`on-watchdog`), graduated restart delay (`RestartSteps=`/`RestartMaxDelaySec=`), `RestartPreventExitStatus=`, `SuccessExitStatus=` (custom clean exit codes and signals), `RemainAfterExit=`, `ExitType=` (main/cgroup); stdout/stderr collection with per-service buffering; 31 exit-handler tests
- ✅ **Exec helper** — `entrypoints/exec_helper.rs`; child process forked by PID 1 that applies all `systemd.exec(5)` directives before exec: mount namespace setup (`ProtectSystem=`, `ProtectHome=`, `PrivateTmp=`, `PrivateDevices=` with synthetic `/dev`, `PrivateNetwork=` with loopback bringup, `PrivateUsers=`, `PrivateMounts=`, `ProtectKernelTunables=`, `ProtectKernelModules=`, `ProtectKernelLogs=`, `ProtectControlGroups=`, `ProtectClock=`, `ProtectHostname=`, `ProtectProc=`, `ProcSubset=`, `ReadOnlyPaths=`, `ReadWritePaths=`, `InaccessiblePaths=`, `BindPaths=`, `BindReadOnlyPaths=`, `TemporaryFileSystem=`); privilege dropping (`User=`, `Group=`, `SupplementaryGroups=`, `DynamicUser=`); capability management (`CapabilityBoundingSet=`, `AmbientCapabilities=`, `NoNewPrivileges=`); seccomp filters (`SystemCallFilter=`, `SystemCallArchitectures=`, `RestrictNamespaces=`, `RestrictRealtime=`, `RestrictSUIDSGID=`, `MemoryDenyWriteExecute=`, `LockPersonality=`, `RestrictAddressFamilies=`); resource limits (all 15 `Limit*=` directives); `OOMScoreAdjust=`, `Nice=`, `IOSchedulingClass=`/`IOSchedulingPriority=`, `UMask=`; directory setup (`StateDirectory=`, `CacheDirectory=`, `LogsDirectory=`, `RuntimeDirectory=`, `ConfigurationDirectory=` with mode control); TTY handling (`TTYPath=`, `TTYReset=`, `TTYVHangup=`, `TTYVTDisallocate=`); stdin routing (`null`, `tty`, `tty-force`, `tty-fail`, `data`, `file`); credential setup (import, load, set, encrypted variants); utmp/wtmp record writing (`UtmpIdentifier=`, `UtmpMode=`); `/dev/kmsg` kernel log backend; 17 tests
- ✅ **Platform abstractions** — `platform/` module; cgroups v1 and v2 (`cgroups/` — process tracking via freezer cgroups, dynamic v1/v2 detection, `move_to_own_cgroup()` for leaf-cgroup compliance); subreaper (`become_subreaper()` via `prctl(PR_SET_CHILD_SUBREAPER)`); eventfd (`EventFd` abstraction using `pipe()` for portable event signaling); privilege dropping (`drop_privileges()` for `User=`/`Group=`); user/group lookup (`pwnam` for passwd queries, `grnam` for group queries); Unix common utilities (file descriptor management, `select(2)` wrappers)
- ✅ **Control interface** — `control/` module; JSON-RPC 2.0 protocol over Unix socket (`/run/systemd-rs/control.socket`) and optional TCP; 35+ commands: `ListUnits`, `ListUnitFiles` (with type filtering and enabled/disabled/static/masked state detection), `ListDependencies` (forward and `--reverse` with box-drawing tree), `Status`, `Show` (with `-p` property filter and `--value` mode), `Cat`, `Start`/`Stop`/`Restart`/`TryRestart`/`ReloadOrRestart`, `Kill` (with signal selection), `Mask`/`Unmask` (symlink management), `Enable`/`Disable`, `IsActive`/`IsEnabled`/`IsFailed`, `ResetFailed`, `LoadNew`/`LoadAllNew`, `Shutdown`, `Suspend`/`Hibernate`/`HybridSleep`/`SuspendThenHibernate`, `ListTimers`, `SetProperty`; `unit_properties.rs` for `show` output formatting; 80 tests
- ✅ **Signal handling** — `signal_handler.rs`; SIGCHLD reaping with `waitpid(WNOHANG)` in a loop; deadlock-free PID table updates via `Arc<Mutex<PidTable>>` (separate from `RuntimeInfo` RwLock to break 3-way deadlock between activation threads, systemctl commands, and exit handler); SIGTERM/SIGINT/SIGQUIT handling for clean shutdown; exit status forwarded to `service_exit_handler` thread for restart policy evaluation
- ✅ **Generator framework** — `generators.rs`; discovers and executes external generators from 6 search paths (`/run/systemd/system-generators/`, `/etc/systemd/system-generators/`, `/usr/local/lib/systemd/system-generators/`, `/usr/lib/systemd/system-generators/`, `/lib/systemd/system-generators/`, plus NixOS package-relative paths); calls each generator with `<normal_dir> <early_dir> <late_dir>` arguments and per-generator timeout; output directories (`/run/systemd/generator/`, `/run/systemd/generator.early/`, `/run/systemd/generator.late/`) inserted into unit search path at correct priority; skips built-in generators (fstab, getty) to avoid conflicts; 16 tests
- ✅ **Unit activation engine** — `unitset_manipulation/activate.rs`; parallel activation of the full dependency graph from the target unit; `collect_unit_start_subgraph()` for dependency collection; `activate_units_recursive()` spawns activation threads respecting `Before=`/`After=` ordering; `activate_unit()` handles all unit types (service, socket, target, mount, swap, timer, path, device, slice); `Condition*`/`Assert*` evaluation before activation; `ActivationSource` tracking (regular vs socket-activation); proper error propagation via `UnitOperationError`
- ✅ **Hardware database** — `hwdb.rs`; reads compiled `hwdb.bin` binary trie (signature `KSLPHHRH`, 80-byte header); path-compressed prefix trie with sorted child arrays (binary search) and shared string table; fnmatch-style glob matching (`*`, `?`, `[...]`) for device modalias lookups; value entries with space-prefixed property keys; `HwdbReader::query()` returns `BTreeMap<String, String>` of matched properties; auto-discovers hwdb.bin from `/etc/udev/`, `/usr/lib/udev/`, and NixOS store paths; used by `systemd-udevd` `hwdb` builtin; 56 tests
- ✅ **Link config** — `link_config.rs`; `.link` file parser for `systemd.link(5)`; `[Match]` section (MACAddress, OriginalName, Path, Driver, Type, Kind, Property, Host, Virtualization, KernelCommandLine, KernelVersion, Credential, Architecture, Firmware); `[Link]` section (Name, NamePolicy, AlternativeNamesPolicy, MACAddressPolicy, MACAddress, MTUBytes, BitsPerSecond, Duplex, AutoNegotiation, WakeOnLan, ReceiveChecksumOffload, TransmitChecksumOffload, GenericSegmentationOffload, LargeReceiveOffload, etc.); loads from 5 priority directories with deduplication by filename; first matching file wins; used by `systemd-networkd` and `systemd-udevd` `net_setup_link` builtin; 76 tests
- ✅ **Runtime state management** — `runtime_info.rs`; `RuntimeInfo` central state struct holding `UnitTable`, `ArcMutPidTable` (deadlock-free via separate `Arc<Mutex<>>`), `FDStore`, `Config`, and eventfds for notification/stdout/stderr/socket-activation signaling; `ArcMutRuntimeInfo` (`Arc<RwLock<RuntimeInfo>>`) passed to all threads; `PidTable` tracks service PIDs with state transitions (`Service`/`Helper`/`ServiceExited`/`HelperExited`)
- ✅ **Logging** — `logging.rs` (fern-based structured logger with colored level output and timestamps) + `kmsg_log.rs` (`KmsgLogger` — `/dev/kmsg` kernel ring buffer backend with syslog priority prefixes for exec helper child processes; graceful fallback to stderr; per-write fd open for `ProtectKernelLogs=` resilience)
