# Phase 2 — Essential System Services

Services required for a fully functional desktop or server:

- 🔶 **`udevd`** — device manager daemon with netlink uevent monitoring (AF_NETLINK / NETLINK_KOBJECT_UEVENT), `.rules` file parser (full syntax: match/assign operators `==`/`!=`/`=`/`+=`/`-=`/`:=`, line continuation, escape sequences, `GOTO`/`LABEL` control flow), property matching (`KERNEL`, `SUBSYSTEM`, `ACTION`, `DRIVER`, `DEVTYPE`, `ATTR{file}`, `ENV{key}`, `RESULT`, `TEST`), parent device traversal (`KERNELS`, `SUBSYSTEMS`, `DRIVERS`, `ATTRS{file}`), assignment actions (`NAME`, `SYMLINK`, `OWNER`, `GROUP`, `MODE`, `ENV{key}`, `TAG`, `RUN{program}`, `RUN{builtin}`, `ATTR{file}`, `SYSCTL{key}`, `OPTIONS`), `IMPORT{program|file|cmdline|builtin|db|parent}`, `PROGRAM` execution with result capture and propagation, udev-style substitution expansion (`$kernel`/`%k`, `$number`/`%n`, `$devpath`/`%p`, `$id`/`%b`, `$driver`, `$attr{file}`/`%s{file}`, `$env{key}`/`%E{key}`, `$major`/`%M`, `$minor`/`%m`, `$result`/`%c` with index, `$name`/`%D`, `$links`, `$root`, `$sys`, `$devnode`/`%N`), glob matching (`*`, `?`, `[...]`, `|` alternatives), device database persistence (`/run/udev/data/`), device tag management (`/run/udev/tags/`), symlink creation/removal in `/dev/`, device node permission setting (OWNER/GROUP/MODE with name resolution), sysfs attribute writing, RUN program execution with environment passing, builtin handlers (path_id, input_id, usb_id, net_id, blkid, kmod, **hwdb**, **keyboard**, **net_setup_link**), **`hwdb` builtin** (reads compiled hardware database `hwdb.bin` from standard search paths `/etc/systemd/hwdb/hwdb.bin`, `/etc/udev/hwdb.bin`, `/usr/lib/systemd/hwdb/hwdb.bin`, `/usr/lib/udev/hwdb.bin` plus NixOS package-relative paths; full binary trie reader with on-disk format parsing — header validation with `KSLPHHRH` signature, trie node/child/value entry deserialization, path-compressed prefix handling, v1 and v2 value entry support; trie traversal with `fnmatch`-style pattern matching — exact character walk with automatic switch to recursive pattern reconstruction and glob matching when `*`/`?`/`[` metacharacters are encountered in node prefixes or child edges; `--subsystem=SUB` parent device traversal walks sysfs hierarchy to find ancestor in requested subsystem and reads its MODALIAS; `--filter=PATTERN` glob filtering on property keys; `--lookup-prefix=PFX` prepends prefix to modalias; `--device=DEVID` alternate source device; positional modalias argument; USB modalias composition from `idVendor`/`idProduct` sysfs attributes matching systemd's `modalias_usb()`; hwdb loaded at daemon startup alongside rules and reloaded on SIGHUP; shared via `Arc<Option<Hwdb>>` with worker threads; `libsystemd::hwdb` module provides `Hwdb` struct with `open()`/`open_default()`/`from_bytes()`/`lookup()`/`get()`, `HwdbBuiltinArgs` parser, `filter_properties()`, and `fnmatch()` glob matcher; 56 unit tests in libsystemd::hwdb + 10 in udevd), **`net_setup_link` builtin** (loads `.link` files from standard search paths via shared `libsystemd::link_config` module, matches network devices against `[Match]` sections, resolves interface names via `NamePolicy=` by checking `ID_NET_NAME_FROM_DATABASE`/`ID_NET_NAME_ONBOARD`/`ID_NET_NAME_SLOT`/`ID_NET_NAME_PATH`/`ID_NET_NAME_MAC` environment variables from earlier builtins, falls back to explicit `Name=`, sets `ID_NET_LINK_FILE` to matching `.link` file path, sets `ID_NET_NAME` for interface renaming, propagates `MTUBytes` as `ID_NET_LINK_FILE_MTU`, `MACAddress` as `ID_NET_LINK_FILE_MACADDRESS`, `MACAddressPolicy` as `ID_NET_LINK_FILE_MACADDRESS_POLICY`, `AlternativeName`/`AlternativeNamesPolicy` as `ID_NET_LINK_FILE_ALTNAMES`), **`keyboard` builtin** (applies `KEYBOARD_KEY_*` scancode→keycode remappings from hwdb properties via `EVIOCSKEYCODE_V2` ioctl on evdev device nodes; comprehensive keycode name→value lookup table with 300+ KEY_*/BTN_* constants from linux/input-event-codes.h; case-insensitive name resolution with numeric decimal/hex fallback; `EVDEV_ABS_*` absolute axis override support with `EVIOCGABS`/`EVIOCSABS` ioctls for min/max/resolution/fuzz/flat fields; ABS_* axis name resolution table; scancode length auto-detection 1/2/4/8 bytes; 35 unit tests), **network interface renaming** via netlink RTM_SETLINK — after rules processing for "net" subsystem "add"/"move" events, renames the interface using `IFLA_IFNAME` if `ID_NET_NAME` (from `net_setup_link` builtin) or `NAME=` (from rules) specifies a different name; also applies MAC address (`IFLA_ADDRESS`) from `ID_NET_LINK_FILE_MACADDRESS` and MTU (`IFLA_MTU`) from `ID_NET_LINK_FILE_MTU`; self-contained NETLINK_ROUTE socket with proper bind/send/recv/ACK handling; MAC address parsing (colon-separated hex); `INTERFACE` env var updated after successful rename so downstream RUN programs and database entries see the final name; graceful error handling for missing ifindex, same-name no-op, invalid MAC format, zero MTU; 29 new unit tests), **event worker thread pool** (concurrent event processing with per-device serialization via `busy_devpaths` tracking — events for the same devpath are queued while earlier events for that device are in-flight, matching real systemd behaviour; configurable `--children-max` concurrency limit; graceful fallback on thread spawn failure), **PROGRAM result capture propagation** (`PROGRAM==` match captures stdout into `program_result`, available to subsequent rules via `$result`/`%c`/`RESULT==`; `PROGRAM=` assignment also captures; event environment passed to PROGRAM commands for full context), **inotify-based rules auto-reload** (`InotifyRulesWatcher` monitors all existing rules directories plus NixOS package-relative paths using `IN_CLOSE_WRITE`/`IN_DELETE`/`IN_MOVED_TO`/`IN_MOVED_FROM`/`IN_CREATE` events; only `.rules` file changes trigger reload; 3-second debounce timer batches rapid changes like package manager installs into a single reload, matching real systemd behaviour; graceful fallback to SIGHUP-only reload if inotify is unavailable; adaptive main-loop poll timeout accelerates during debounce; 8 unit tests), event queue with settle support, control socket, sd_notify protocol (READY/WATCHDOG/STATUS/STOPPING), signal handling (SIGTERM/SIGINT/SIGHUP/SIGCHLD); `udevadm` CLI with `info` (query by name/path, property export, attribute walk, database export, cleanup, `--device-id-of-file` for block device identification), `trigger` (action/subsystem/attribute/property/tag/sysname filters, prioritized subsystems, dry-run, verbose), `settle` (timeout, exit-if-exists, **inotify-based queue file watching** — watches `/run/udev/` for queue file deletion via inotify with 50ms poll for responsive settle detection, falls back to 200ms polling if inotify unavailable; 6 new unit tests), `monitor` (kernel uevent listening with subsystem filter, property display), `test` (device property display with rules file enumeration), `control` (reload/ping/stop/start exec queue, exit, log level, children max), `test-builtin`, `version`; 254 unit tests (215 udevd + 39 udevadm); missing: `udev` database locking, D-Bus interface
