56 unit tests in libsystemd::hwdb + 10 in udevd), **`net_setup_link` builtin** (loads `.link` files from standard search paths via shared `libsystemd::link_config` module, matches network devices against `[Match]` sections, resolves interface names via `NamePolicy=` by checking `ID_NET_NAME_FROM_DATABASE`/`ID_NET_NAME_ONBOARD`/`ID_NET_NAME_SLOT`/`ID_NET_NAME_PATH`/`ID_NET_NAME_MAC` environment variables from earlier builtins, falls back to explicit `Name=`, sets `ID_NET_LINK_FILE` to matching `.link` file path, sets `ID_NET_NAME` for interface renaming, propagates `MTUBytes` as `ID_NET_LINK_FILE_MTU`, `MACAddress` as `ID_NET_LINK_FILE_MACADDRESS`, `MACAddressPolicy` as `ID_NET_LINK_FILE_MACADDRESS_POLICY`, `AlternativeName`/`AlternativeNamesPolicy` as `ID_NET_LINK_FILE_ALTNAMES`), **`keyboard` builtin** (applies `KEYBOARD_KEY_*` scancode→keycode remappings from hwdb properties via `EVIOCSKEYCODE_V2` ioctl on evdev device nodes; comprehensive keycode name→value lookup table with 300+ KEY_*/BTN_* constants from linux/input-event-codes.h; case-insensitive name resolution with numeric decimal/hex fallback; `EVDEV_ABS_*` absolute axis override support with `EVIOCGABS`/`EVIOCSABS` ioctls for min/max/resolution/fuzz/flat fields; ABS_* axis name resolution table; scancode length auto-detection 1/2/4/8 bytes; 35 unit tests), **network interface renaming** via netlink RTM_SETLINK — after rules processing for "net" subsystem "add"/"move" events, renames the interface using `IFLA_IFNAME` if `ID_NET_NAME` (from `net_setup_link` builtin) or `NAME=` (from rules) specifies a different name;
