# Phase 1 — Core System (PID 1 + systemctl + journald)

The minimum viable system to boot a real Linux machine:

- ✅ **`systemd` (PID 1)** — service manager with all core unit types (service, socket, target, mount, timer, path, slice, scope) and all service types (`simple`, `exec`, `notify`, `notify-reload`, `oneshot`, `forking`, `dbus`, `idle`), default target handling, parallel activation, fstab generator, getty generator, NixOS early boot setup, full `Condition*`/`Assert*` directive support (15 check types), proper `Type=idle` deferral (idle services wait for all other jobs to complete before starting); **emergency/rescue mode** — kernel command line parsing (`/proc/cmdline`) for boot target override: `systemd.unit=<target>` (explicit override, last occurrence wins), `emergency` keyword → `emergency.target`, `rescue`/`single`/`s`/`S`/`-s`/`-S` → `rescue.target`, SysV runlevel compat (`1` → rescue, `2`/`3`/`4` → multi-user, `5` → graphical), `rd.systemd.unit=` recognized and ignored for real root; boot mode banner on console for emergency/rescue/custom targets; 25 unit tests; **path unit support** with `PathExists=`, `PathExistsGlob=`, `PathChanged=`, `PathModified=`, `DirectoryNotEmpty=`, `MakeDirectory=`, `DirectoryMode=`, `TriggerLimitIntervalSec=`, `TriggerLimitBurst=`, `Unit=` — **inotify-based filesystem monitoring** with instant notification when watched paths change (per-condition watch strategy: `PathExists=`/`PathExistsGlob=` watch parent directory for `IN_CREATE`/`IN_MOVED_TO`; `PathChanged=` watches path for `IN_ATTRIB`/`IN_CLOSE_WRITE`/`IN_CREATE`/`IN_DELETE`/`IN_MOVED_FROM`/`IN_MOVED_TO`/`IN_DELETE_SELF`/`IN_MOVE_SELF`; `PathModified=` watches path for `IN_CLOSE_WRITE`/`IN_DELETE_SELF`/`IN_MOVE_SELF`; `DirectoryNotEmpty=` watches directory for `IN_CREATE`/`IN_MOVED_TO`), automatic parent-directory watching for non-existent paths with upgrade to direct watch when the target appears, `PathInotify` watch manager with multi-entry-per-WD support (handles inotify WD reuse when multiple conditions watch the same path), `poll(2)` on the inotify fd with 500ms timeout for responsive event handling, periodic full poll-based safety-net scan every 10 seconds (catches network filesystem events and lost watches), graceful fallback to pure 2-second polling when inotify is unavailable, `IN_IGNORED` event handling for automatic watch re-establishment, dynamic watch synchronization (adds watches for newly activated path units, removes watches for deactivated ones), glob matching, rate limiting, and `MakeDirectory=` support; 44 unit tests; **full sd_notify protocol**: `MAINPID=` updates tracked PID, `WATCHDOG=1` pings recorded, `WATCHDOG=trigger` immediate action, `RELOADING=1`/`STOPPING=1` state tracking with `READY=1` reload-complete detection, `WATCHDOG_USEC=` dynamic timeout override, `ERRNO=`/`BUSERROR=`/`EXIT_STATUS=` error reporting stored for status display, `MONOTONIC_USEC=`/`INVOCATION_ID=` metadata, **`FDSTORE=1`/`FDSTOREREMOVE=1`/`FDNAME=`** fd store with SCM_RIGHTS via `recvmsg()` (respects `FileDescriptorStoreMax=`, passes stored FDs back on restart via `LISTEN_FDS`/`LISTEN_FDNAMES`); **watchdog enforcement**: background thread periodically scans running services with `WatchdogSec=` configured, detects unresponsive services that have not sent `WATCHDOG=1` within the timeout (supports `WATCHDOG_USEC=` dynamic override, `WATCHDOG=trigger` immediate timeout, `WatchdogSignal=` custom signal defaulting to SIGABRT), kills timed-out services and sets `watchdog_timeout_fired` flag so the exit handler correctly applies `Restart=on-watchdog`/`Restart=on-failure`/`Restart=on-abnormal` policies; watchdog state (`watchdog_last_ping`, `watchdog_timeout_fired`, `watchdog_usec_override`) cleared on service (re)start; `READY=1` initializes the watchdog reference timestamp for notify-type services; non-notify services start the watchdog clock from process fork; **exec command line prefixes**: all 6 prefixes supported — `-` (ignore failure), `@` (argv[0] override), `+` (full privileges — skips all namespace/sandbox/security restrictions and privilege drop), `!` (elevated privileges for SUID binaries), `!!` (elevated privileges with file capabilities), `:` (clean environment — strips all configured env vars, keeps only PATH/NOTIFY_SOCKET/LISTEN_FDS/LISTEN_FDNAMES); prefixes combinable in any order on ExecStart=/ExecStartPre=/ExecStop=/ExecStopPost= lines; `+`/`!`/`!!` set `privileged_prefix` on ExecHelperConfig to bypass ProtectSystem=, ProtectHome=, PrivateTmp=, PrivateDevices=, PrivateNetwork=, ProtectHostname=, CapabilityBoundingSet=, NoNewPrivileges=, LockPersonality=, User=/Group= privilege drop; `:` sets `clean_environment` to discard Environment=/EnvironmentFile=/PassEnvironment= variables; **transient units** — in-memory service units created via `start-transient` control protocol command (used by `systemd-run`), supports `unit` (name), `command` (ExecStart), `description`, `user`, `group`, `working_directory`, `type` (service type), and `remain_after_exit` parameters; transient units are inserted into the unit table with lenient dependency wiring and started immediately; **daemon re-execution** — `daemon-reexec` control command and `SIGRTMIN+13` signal trigger `execve()` of the service manager binary in-place, serializing running-service PID state to `/run/systemd/systemd-rs-reexec-state` (unit_name/PID/ServiceType per line) and restoring PID tracking on the new instance (verifies processes are still alive via `kill(pid, 0)`, re-inserts into PID table with correct ServiceType); environment variable `SYSTEMD_RS_REEXEC=1` marks reexec mode; **`SIGRTMIN+` real-time signals** — 16 signal offsets registered via `signal_hook` (computed from `libc::__libc_current_sigrtmin()`): `+0` switch to `default.target`, `+1` `rescue.target`, `+2` `emergency.target`, `+3` halt, `+4` poweroff, `+5` reboot, `+6` kexec, `+13` daemon-reexec, `+14` log system status summary, `+15`/`+22` set log level to debug, `+16`/`+23` set log level to info, `+20`/`+21` enable/disable show-status (placeholder), `+24` immediate exit (container mode); target-switching signals spawn a thread to activate the requested target and its dependencies; 24 new unit tests
- ✅ **`systemctl`** — CLI including `start`, `stop`, `restart`, `try-restart`, `reload-or-restart`, `enable`, `disable`, `kill`, `reset-failed`, `suspend`, `hibernate`, `hybrid-sleep`, `suspend-then-hibernate`, `status`, `show`, `cat`, `list-units`, `list-unit-files`, `list-dependencies`, `mask`, `unmask`, `is-active`, `is-enabled`, `is-failed`, `poweroff`, `reboot`, `daemon-reload`, `daemon-reexec`, `edit`, `set-property`, `revert`; `show` supports `-p`/`--property` filtering and `--value` for value-only output; `cat` displays unit file source with path header; `list-dependencies` shows dependency tree with box-drawing characters, status markers, `--reverse` for reverse deps; `mask`/`unmask` create/remove `/dev/null` symlinks in `/etc/systemd/system/`; `kill` supports `--signal`/`-s` with numeric or named signals; `reset-failed` clears error state for one or all units; sleep commands (`suspend`/`hibernate`/`hybrid-sleep`/`suspend-then-hibernate`) spawn `systemd-sleep` via PID 1; `daemon-reexec` triggers full re-execution of the service manager binary (separate from `daemon-reload` which only reloads unit files); **`edit`** — client-side unit file editing with drop-in override support (`/etc/systemd/system/<unit>.d/override.conf`), `--full` mode for full unit file copy, editor selection via `$SYSTEMD_EDITOR`/`$EDITOR`/`$VISUAL`/`vi`, automatic `daemon-reload` after editing, template comment stripping, empty-override cleanup; **`set-property`** — runtime property assignment with persistent drop-in file creation (`/etc/systemd/system/<unit>.d/50-set-property.conf`), automatic section grouping (unit-level properties under `[Unit]`, type-specific properties under `[Service]`/`[Socket]`/etc. inferred from unit suffix), multi-property support; **`revert`** — removes all local customizations for a unit: `/etc/systemd/system/<unit>.d/` drop-in directory, `/run/systemd/system/<unit>.d/` runtime drop-ins, admin override files (only when vendor copy exists in `/usr/lib/` or `/lib/`), runtime overrides; handles common flags (`--no-block`, `--quiet`, `--force`, `--no-pager`, `--system`, `-a`, `-q`, `-f`, `-l`, `-t`, `-p`); proper exit codes for query commands; 20 new unit tests
- ✅ **`journald`** — journal logging daemon with `/dev/log` socket, native protocol, syslog protocol, kernel `kmsg`; **rate limiting** — per-source (unit/identifier/PID) token-bucket rate limiter with configurable `RateLimitIntervalSec=`/`RateLimitBurst=`, suppression counting with automatic journal summary entry on window reset ("Suppressed N messages from X"), `RateLimitIntervalSec=0` or `RateLimitBurst=0` disables rate limiting, periodic GC of stale rate-limit state; **journal file rotation** — automatic size-based rotation (when active file exceeds `SystemMaxFileSize=`), time-based rotation via `MaxFileSec=` (maintenance thread checks active file age against configurable max time span, default 1 month), SIGUSR2-triggered manual rotation, active file creation time tracking; **disk usage limits** — `SystemMaxUse=`/`RuntimeMaxUse=` total disk usage caps enforced by vacuum (oldest files removed first), `MaxFiles=` file count limit, `SystemKeepFree=`/`RuntimeKeepFree=` minimum free disk space enforcement via `statvfs(2)` (vacuums oldest files when available space drops below threshold, default 4 GiB), periodic vacuum in maintenance thread (every 60 s), hourly disk usage logging with warning journal entry when usage exceeds 80% of limit; **forward-secure sealing** — `Seal=yes/no` configuration, HMAC key-chain with forward secrecy (key advanced via deterministic mixing after each seal epoch, old key material zeroed), periodic seal entries appended to journal with `_JOURNAL_SEAL_TAG=` (hex-encoded tag) and `_JOURNAL_SEAL_EPOCH=` fields, configurable seal interval (default 15 min); **wall message forwarding** — `ForwardToWall=yes` broadcasts high-priority messages (respects `MaxLevelWall=`) to all logged-in terminals: utmp-based terminal discovery (`/var/run/utmp` USER_PROCESS entries) with fallback to `/dev/pts/*` + `/dev/tty[0-9]*` enumeration, `O_NOCTTY|O_NONBLOCK` non-blocking writes, messages include timestamp, priority label, identifier, and PID; 50 unit tests
- ✅ **`journalctl`** — journal query tool with comprehensive filtering and output formats; **filters**: `--boot`/`-b` (boot ID/offset), `--unit`/`-u` (systemd unit), `--user-unit` (user unit), `--identifier`/`-t` (syslog identifier), `--priority`/`-p` (single or range, by name or number), `--facility` (syslog facility 0-23 or name: kern, user, mail, daemon, auth, syslog, lpr, news, uucp, cron, authpriv, ftp, local0-7), `--transport` (journal, syslog, kernel, stdout, audit, driver), `--grep`/`-g` with `--case-sensitive`, `--since`/`--until` (ISO 8601, "today"/"yesterday"/"now"/"tomorrow", relative "-1h"/"-30min"/"-2d", UNIX timestamps), `--pid`/`--uid`/`--gid`, `--dmesg`/`-k` (kernel messages), `--system`/`--user` (system vs user journal entries), `--machine`/`-M` (filter by machine ID or hostname), `--cursor`/`--after-cursor`, free-form `FIELD=VALUE` match expressions; **output formats** (`-o`): `short` (syslog-style timestamp), `short-full` (year-inclusive "Day YYYY-MM-DD HH:MM:SS TZ"), `short-iso` (ISO 8601), `short-iso-precise` (ISO 8601 with microseconds), `short-precise` (syslog with microsecond precision), `short-monotonic` (monotonic seconds since boot), `short-unix` (UNIX epoch seconds with microsecond fractional), `with-unit` (short format with unit name column), `verbose` (all fields indented), `json` (compact JSON), `json-pretty` (indented JSON), `json-sse` (Server-Sent Events: `data: <json>\n\n`), `json-seq` (RFC 7464 JSON Text Sequences with RS 0x1E prefix), `cat` (message only), `export` (systemd journal export format); **output control**: `--output-fields` (comma-separated field whitelist for verbose/json/export modes, metadata `__`-prefixed fields always preserved), `--no-hostname` (suppress hostname in short formats), `--utc`, `--all`/`-a`, `--show-cursor`, `--no-pager`, `--quiet`/`-q`, `--catalog`/`-x`; **follow mode** (`-f`/`--follow`): 500ms polling with `--no-tail` to show all matching entries before following (default tails last 10), `--lines`/`-n` limit, `--reverse`/`-r`; follow-mode filters mirror all main query filters (unit, user-unit, identifier, priority, facility, transport, system/user, machine, PID, UID, GID, grep, free-form matches); **commands**: `--disk-usage` (human-readable journal size), `--list-boots` (boot offset/ID/time-range table), `--field`/`-F` (unique field values), `--fields`/`-N` (all field names), `--header` (file path, file count, disk usage, machine ID, boot ID, entry count, first/last timestamps), `--flush` (SIGUSR1 to journald), `--rotate` (SIGUSR2 to journald), `--verify`, `--vacuum-size` (remove oldest files until below K/M/G/T limit), `--vacuum-time` (remove files older than time span), `--vacuum-files` (remove files until count limit); `--directory`/`-D` and `--file` for custom journal paths; 59 unit tests
